1. github 이란 
github은 스프트웨어 개발 프로젝트 소스코드를 버전관리 관리 서비스이다. 버전 관리를 하면서 
소스코드 열람 간단한 버긍관리, SNS 소통기능으로 협업이 필수 요소가 된 개발자들에게 있어 없어서은 안될 서비스 입니다.
github을 사용해 버전 관리를 하는 기업도 다수 있다.

2. github 계정등록
github 페이지를 들어가 Sing up을 누르고 회원가입을 하면된다. 도중 나오는 유형중 무료로 사용할 수 있는 Free를 
선택하면 된다.

3. github 사용을 위한 사전 지식
 (1) 용어 개념 
	커밋(commit) : 파일을 추가하거나 변경 냐용을 로컬 저장소에 저장하는 작업
	푸시 (push) : 파일을 추가하거나 변경 내용을 원격 저장소에 업로드 하는 작업

 (2) 로컬 저장소와 원격 저장소
	저장소는 파일이나 디렉토리를 저장하는 장소 입니다.
	 변경이력을 관리하고자 하는 디렉토리 등을 저장소의 관리하에 두는 것으로 디렉토리에 있는 파일 등의 변경이력을 기록할 수 있다. 
	(변경 이력을 기록할 때는 push)

(3) 브랜치 (branch)
	버전의 유지 보수를 하며 새로운 기능을 추가하고 버그를 수정하는데 사용 한개의 master 이라는 메인 장소에서 분기를 나누어
	자신만의 기록을 해가는 것이다. 나누어진 분기는 서로에게 영향을 받지 않기 떄문에 같은 저장소에서 개발을 해나갈 수 있다.

4. github 사용법 
	1. githupdp 저장소 작성(git init):자기가 저장소를 만들거나 아니면 클론(git clone): 다른 사람이 만든 저장소의 내용을 복제 하던 가

	2. 파일을 작성, 편집 => 파일의 수정 정보를 git 인덱스에 추가 (git add)

	3. 변경 결과를 로컬저장소에 커밋(git commit)

	4. 로컬 저장소를 푸쉬(push) 해서 원격 저장소에 반영(git push)

	5. 다음에는 병합을 위한 준비 먼저 push한 내용을 보고 상의 하고

	6. master 를 pull 해서 최신 버전에 병합(marge)한다. 
	(다시 브렌치를 만들어 나눈다음 2번 부터 반복하면 된다.)
